Ready to Install! Let's Go!

This user guide, will help you in a few steps to setup and use HMS easily

After successfully installed the software, you will have to make this simple configuration at first start.

Step 1
Configure file location

As the HMS is built to run into a windows server and use a file system as database you will have to determine where you want to save that **important** data on the admin computer.

Step 2
Set Super-user Account

You have to add a super-user, you will have to provide a name for the account Owner, a username with an password. the super-user is the first granted user recognize by the system with full privilege over all modules and features.

As first user, use the Super-user's credential data you just created

Each time you will launch the HMS, you will have to submit your credential

All features will set disabled until you provide a valid username and password

HMS propose a greatfull UX through an intuitive UI with a lot of tools!

  • 1- Menu bar and features
  • 2- User panel button, you can perform a right-click over this button for logging out
  • 3- Alert panel - after being read a alert message will automatically desappear
  • 4- Quick access link
  • 5- Mailbox preview - the small icon in top right on this panel give you access to the mailbox
  • 6- Task overview tab
  • 7- Themes controller options
  • 8- Productivity panel

At this point we going to introduce you one by one each features!

The Staff Management
Patient Files

Each patient record is represented by a file named with that patient's file number. By double-clicking on a folder, you can access it. The files are prioritized so that each folder contains subfolders containing data about the patient's health status.

Consultation

The consultation window allows you to save relevant data on the state of health of your patients, ranging from signs to diagnoses made by the consulting doctor, as well as the prescription.

**If you wish to hospitalize a patient after consultation, simply check the “Hospitalization needed” box before saving

Hospitalisation

HMS offers the possibility of monitoring the progress of each patient's hospitalization. monitored his treatment and his state of health.

Maternity

The Delivery Information panel allows you to save data on a delivery in progress or completed

By default the delivery status is initialized to Awaiting. in this case, the buttons of the New Born panel are disabled

  • To be eligible for maternity care, the patient must be on the list of hospitalized patients and be female. Once these constraints have been raised, the patient's file number will appear in the drop-down menu above.

  • To admit a new patient into martenity list
  • To completely clear the forms
  • To remove a patient from martenity list
Insurance

Managing insurance services allows you to pay patient expenses and produce invoices that take into account the costs covered by the companies.

This module is not responsible for managing all of the services that can be offered by an insurance company, however it is involved in regulating costs in cases of hospitalization, consultation, treatment, billing and more.

Reports
Rooms

The room module was created to allow the management of rooms, rooms and the equipment available in the hospital.

This module plays an important role in billing, thanks to the precision it gives on resource utilization and associated costs

Users

To define the privileges on a module or an accessibilities. All you have to do is right-click on the three-point icon and choose the appropriate option from the context menu that appears.

  • Indicates that you have full privileges over this module
  • Indicates that you only allow to write
  • Indicates that you only allow to read

the user module allows you to manage accounts, privileges, accessibilities, authentication parameters as well as the creation of new services in the hospital.

Depending on your access rights, certain functionalities of this module will not be accessible to you, the LOCK icon at the top right of each window defines your privileges on a given module. This privilege in itself is predefined during the creation of each account by a legal administrator.

the module is divided into two main panels, the first panel, the USER Panel gives information relating to the current user in case the creation of a new user is not in progress

It is possible for the logged in user if they have the right to modify their password through this panel.

The second panel located underneath contains the accessibilities, modules and preferences to define for a user. they cannot be modified by an average user
